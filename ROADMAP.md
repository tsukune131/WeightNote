# WeightNote 公開ロードマップ

ゴール: App Storeでの一般公開。
方針: **無料でできる改善をすべて先に行い、納得してから課金ステップ(Apple Developer 年99ドル)に進む。**
各フェーズの最後に「チェックポイント」があり、そこで次に進むか判断する。

---

## フェーズA: アプリ品質の底上げ(費用: 0円)

PWAのまま、毎日使って気持ちいいレベルまで磨く。

- [x] A-1 食事入力の補助 (2026-07-19)
  - マイメニュー登録(よく食べる物を名前+kcalで保存、ワンタップ入力)
  - よく使う順に自動で並ぶ(使用回数を記録)
- [x] A-2 継続を促す仕掛け (2026-07-19)
  - 連続記録日数(ストリーク)表示
  - 直近7日間のサマリー(貯金合計・体重変化・目標達成日数)
- [x] A-3 オンボーディング (2026-07-19)
  - 初回起動時の案内フロー(ようこそ→プロフィール作成→目標設定→使い方ガイド)
- [ ] A-4 パフォーマンス改善
  - グラフ画面の遅延読み込みでバンドル分割(初回表示の高速化)
- [ ] A-5 デザイン磨き込み
  - アイコン・配色・余白・画面遷移の見直し、ダークモード微調整

**チェックポイントA**: 家族でPWAを1〜2週間使い、毎日の記録が苦にならないか確認。

## フェーズB: 公開に必要な土台(費用: 0円)

お金をかけずに作れる「ストア品質の裏側」を先に済ませる。

- [ ] B-1 クラウド同期 + アカウント(Firebase無料枠)
  - ログイン(メール or Google)
  - 端末をまたいだデータ同期、機種変更対応
  - 家族でお互いの推移を見られる共有(任意設定)
- [ ] B-2 Capacitorプロジェクト化(Windows上でできる範囲)
  - iOSプロジェクト生成、Webビルドの組み込み確認
  - HealthKit・ローカル通知のプラグイン選定と実装(ビルドは後)
- [ ] B-3 法務文書の準備
  - プライバシーポリシー、利用規約、健康アドバイス免責の文面作成と公開ページ
- [ ] B-4 エラー監視(Sentry無料枠)導入

**チェックポイントB**: 同期込みのPWAで納得できるか確認。
**ここが課金判断ポイント** — 納得したらApple Developer登録(年99ドル)へ。

## フェーズC: TestFlight配布(費用: 年99ドル)

- [ ] C-1 Apple Developer Program登録(ユーザー作業・年99ドル)
- [ ] C-2 クラウドビルド環境(Codemagic無料枠)で証明書設定とiOSビルド

- [ ] C-4 ローカル通知(記録リマインダー)の実機確認
- [ ] C-5 TestFlightで家族に配布、実機フィードバック反映

**チェックポイントC**: 市販アプリと比べて遜色ないか。家族の継続利用で判断。
